                               Executive Summary
This Business Requirements Document defines the requirements for the Doon: Smart Regional Activity Tracking and Recommendation System  --  an AI-powered web and mobile platform for the CALABARZON region developed at Batangas State University ARASOF-Nasugbu, TNEU. The project is managed by Asst. Prof. Benjie R. Samonte and sponsored by Asst. Prof. Arlene M. Mendoza. The platform serves three user roles: Tourist, Local (service providers and municipalities), and Admin. The project runs from February 12, 2026 to May 15, 2026, across five WBS phases: Initiation (1.1), Planning (1.2), Execution (1.3), Control (1.4, concurrent), and Closeout (1.5), producing 18 formal deliverables. The system is built on a confirmed tech stack of Laravel (backend), React (web frontend), Flutter (mobile), PostgreSQL (database), and Figma (UI/UX design).

                                Needs Statement
Tourism data across the five provinces of CALABARZON (Batangas, Laguna, Cavite, Rizal, and Quezon) is fragmented across social media, travel blogs, and informal channels with no unified source for Tourists, Local service providers, or LGUs. Trip planning is inefficient and regional tourism potential is underutilized.

The Doon project resolves this by building a single AI-powered web and mobile platform with a multi-criteria recommendation engine, an AI chatbot, an interactive regional map, smart travel features (weather integration, itinerary planning, packing suggestions), a local business directory, and an LGU analytics dashboard  --  giving all three user roles a shared, reliable system that promotes sustainable regional tourism development aligned with SDG 8, 9, 11, and 12.

                              Project Constraints
                                Constraint Type
                                  Description
                             Mitigation / Response
Time
Project must close by May 15, 2026 (Gantt closeout)  --  all 18 deliverables have fixed Gantt Chart due dates; delays cascade through predecessor-linked tasks.
Strict milestone-based schedule with deliverables tied to predecessor tasks; no scope changes permitted after Kick Off.
Team Availability
Fixed five-member team  --  absence of any member delays dependent deliverables; no backup personnel identified.
Tasks distributed by confirmed roles; weekly check-ins and Gantt Chart adherence maintained throughout all phases.
Data Quality
LGU tourism data accuracy not guaranteed  --  incomplete or inaccurate records may impact destination listing quality and recommendation accuracy.
Data validation rules built into the platform; LGU partners engaged early in Phase 1.1 to confirm data availability.
Technology
Third-party API availability and pricing  --  geolocation and weather API rate limits and licensing costs may constrain real-time features.
Fallback plans required; free-tier APIs prioritized (OpenWeatherMap); Google Maps usage capped to stay within ₱1,500 budget.
Scope
No changes after Kick Off (Feb 26, 2026)  --  any scope change requires formal review and written approval from Asst. Prof. Arlene M. Mendoza.
All out-of-scope requests formally documented and deferred; change requests escalated to Project Manager immediately.
Budget
Institutional academic budget only  --  cost overruns must be escalated immediately; no discretionary budget available.
Open-source stack (Laravel, React, Flutter, PostgreSQL) eliminates licensing costs; only API and hosting fees are paid.
Data Privacy
Data Privacy Act of 2012 compliance required  --  user location and behavioral data collection requires explicit consent; role-based access and secure authentication must be implemented.
Role-based access control, data encryption, explicit consent prompts, and audit logging implemented as core system features.
